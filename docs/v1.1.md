### 数据库设计

新增 question_display_rules 表，用于存储问题级联显示规则。
规则说明：
- 被控制的问题默认不显示（前端隐藏）。
- 当触发问题（trigger_question_id）的某个或某些选项（trigger_option_ids）被选中时，被控制的问题才会显示。
- 支持多触发选项、多条件组合（与/或）。
- 一个被控制的问题可对应多条规则，多条规则之间为“或”关系，任意一条满足即显示。

表结构如下：
```sql
CREATE TABLE show_rules (
    id INT AUTO_INCREMENT PRIMARY KEY,
    trigger_question_id INT,
    trigger_option_ids JSON,
    show BOOLEAN DEFAULT TRUE,
    create_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (question_id) REFERENCES questions(id)
);
```

评分题增加选项配置（直接使用question_options表），用于存储评分题的选项及分值。
如果配置了选项及分值，前端在渲染评分题时，会根据配置显示选项及分值。
如果未配置选项及分值，前端默认显示1-5分的选项。
